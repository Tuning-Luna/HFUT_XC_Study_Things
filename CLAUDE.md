## 项目概述

**HFUT XC Study Things** — 合肥工业大学宣城校区计算机科学与技术（CS）专业本科学习资料仓库。涵盖大一至大三的全部课程资料（实验报告、课程设计、作业、考试真题、课堂PPT、教材、项目代码），通过 GitHub Pages 网站分发：<https://tuning-luna.github.io/HFUT_XC_Study_Things>

## 仓库结构

课程目录按 **年级前缀** 分组排序：

| 前缀 | 说明 |
|------|------|
| `【公共课】*` | 公共通识课程（思政、英语、题库） |
| `【大一】*` | 大一课程 |
| `【大二】*` | 大二课程 |
| `【大三】*` | 大三课程 |

每个课程目录内部遵循松散一致的结构：
- `README.md` — 课程说明（教师、考试建议等）
- `实验/` — 实验任务书、报告、代码
- `课设/` — 课程设计项目
- `考试/` 或 `真题/` 或 `试卷/` — 历年考试真题
- `作业/` — 课后作业及解答
- `PPT/` 或 `上课PPT与教材/` — 课堂幻灯片
- `复习资料/` — 复习笔记总结
- `Zips/` — 大文件的压缩归档

## 网站构建

### 技术栈
- **MkDocs** + **mkdocs-material** 主题构建 GitHub Pages 站点
- 自动部署 CI: `.github/workflows/gh-pages.yml`

### 关键脚本
- `gen-md.py` — 遍历课程目录，为每门课生成 `docs/<课程名>.md`，包含文件列表（链接到 GitHub Raw 内容）
- `mkdocs.yml` — 站点配置（Material 主题、亮暗色模式切换）

### 本地预览
```bash
pip install mkdocs mkdocs-material
python gen-md.py
cp README.md ./docs/index.md
cp -r assets ./docs/assets
mkdocs serve
```

### CI/CD
每次推送至 `main` 分支自动触发 GitHub Actions：
1. 安装 Python + MkDocs
2. 运行 `gen-md.py` 生成文档页面
3. 执行 `mkdocs gh-deploy --force` 发布到 `gh-pages` 分支

## 如何添加新资料

### 目录命名规范

```
【年级】课程名/
├── PPT/             # 课堂幻灯片（可标注新旧版本）
├── 实验/            # 实验任务书、实验代码、实验报告（按实验编号整理）
├── 课设/            # 课程设计项目（含代码、报告）
├── 考试/ 或 真题/   # 历年考试真题（含答案更佳）
├── 作业/            # 课后作业及解答
├── 复习资料/        # 复习笔记、总结
├── 教材and解析/     # 教材 PDF、课后习题答案解析
├── Zips/            # 大文件压缩包
└── README.md        # 课程说明（教师、考试形式、备考建议）
```

### 描述内容的行文规范

在 `README.md` 的资料目录表格中更新课程内容时，遵循以下原则：

1. **具体化** — 不用笼统的"PPT，实验，真题"，而是写明具体情况：
   - PPT 有几套？新旧？多少份？如 `PPT（新旧两套 21 份）`
   - 实验有多少个？用什么语言？如 `实验 8 次（Verilog/SystemVerilog 40+ 文件）`
   - 真题覆盖什么年份范围？如 `真题（合肥+宣城两校区 2012-2024 共 40+ 份）`
   - 有什么特色资源？如 `吴恩达课程，30+ 本电子书`

2. **突出亮点** — 某课程特别丰富的资源要体现出来（如"猴博士课堂笔记"、"LaTeX 源码复习笔记"、"AsmTools 工具包"）

3. **标明教材版本** — 如有教材，尽量写明书名和版本，如 `邱关源电路第 5 版`、`Java 2 实用教程第 5 版`

4. **注明编程语言/工具** — 实验和课设代码注明语言：`C++`、`Java`、`Python/Jupyter`、`Verilog`、`MIPS 汇编`

5. **描述结构化的课程** — 如一门课分多个阶段（如电路分析+模电两部分），在描述中用 `+` 分隔

### 更新资料目录后的后续操作

1. 如果新增了课程目录，需要重新运行 `gen-md.py` 生成对应的文档页面
2. 推送至 `main` 后，CI 会自动重新部署网站
3. 如果要添加大四 `【大四】*` 课程，前缀格式与现有保持一致

## 添加新资料完整操作流程

每次向仓库添加了新的课程资料后，按以下步骤使网站自动同步：

### 第一步：添加文件到对应课程目录

直接将新资料放入对应的课程目录中。目录结构参考上面的规范：

```
【年级】课程名/
├── 实验/     → 新实验文件放这里
├── 课设/     → 新课设文件放这里
├── 考试/     → 新真题试卷放这里
├── 作业/     → 新作业文件放这里
├── Zips/     → 超大文件压缩包放这里（推荐超过 50MB 的文件）
└── ...
```

> 大文件（>50MB）建议压缩后放入 `Zips/` 目录，不直接上传散文件到根目录，也方便同学按需下载。

### 第二步：更新 README.md 资料描述表格

打开对应课程目录下的 `README.md`（如果没有则新建），按照**描述内容的行文规范**（见上）更新表格中该课程的描述。重点关注：
- 更新文件数量和覆盖范围（如"新增 2026 年真题 2 份"）
- 如新增了显著内容，同步更新仓库根目录 `README.md` 的总表

### 第三步：更新 mkdocs.yml 导航（仅新增课程时需要）

如果是新增了一门**从未有过**的课程：
1. 在 `mkdocs.yml` 的 `nav:` 中对应年级下添加条目
2. 格式：`- 课程名: 【年级】课程名.md`
3. 注意保持年级内的课程顺序

### 第四步：重新生成本地预览（可选）

在仓库根目录运行：
```bash
python gen-md.py                    # 为所有课程重新生成 docs/*.md
cp README.md ./docs/index.md        # 首页同步
cp -r assets ./docs/assets          # 静态资源
mkdocs serve                        # 本地预览 http://127.0.0.1:8000
```

### 第五步：提交并推送

```bash
git add .
git commit -m "添加 XXX 课程资料（XXX 新增内容说明）"
git push
```

推送至 `main` 后，GitHub Actions CI 会自动执行：
1. 安装 Python + MkDocs
2. 运行 `gen-md.py` 生成文档页面
3. 执行 `mkdocs gh-deploy --force` 发布到 `gh-pages` 分支
4. 网站（https://tuning-luna.github.io/HFUT_XC_Study_Things）会在几分钟内更新

### 新增内容的特别提醒

| 场景 | 注意事项 |
|------|----------|
| **新增课程目录** | 必须更新 `mkdocs.yml` 的 `nav:`；运行 `gen-md.py`；更新根目录 `README.md` 总表 |
| **往已有课程加文件** | 只需 `git push`，CI 会自动重新生成网站文件列表 |
| **新增 Zips 压缩包** | 注意仓库单文件限制（GitHub 建议 <50MB，硬限制 100MB） |
| **删除或重命名文件** | 网站会自动反映删除，但不会删除已缓存的旧页面；如要彻底清理需先 `git rm` 再推送 |
| **新增课程时顺便写 README** | 课程 `README.md` 的内容会自动出现在网站对应页面的文件列表上方，包含教师、考试形式、备考建议等信息 |
