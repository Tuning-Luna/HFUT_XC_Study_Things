## 项目概述

**HFUT XC Study Things** — 合肥工业大学宣城校区计算机科学与技术（CS）专业本科学习资料仓库。涵盖大一至大三的全部课程资料（实验报告、课程设计、作业、考试真题、课堂PPT、教材、项目代码），通过 GitHub Pages 网站分发：<https://tuning-luna.github.io/HFUT_XC_Study_Things>

## 仓库结构

课程目录按 **年级前缀** 分组排序：

| 前缀 | 说明 |
|------|------|
| `【公共课】*` | 公共通识课程（思政、英语、题库） |
| `【大一】*` | 大一课程 |
| `【大二】*` | 大二课程 |
| `【大三】*` | 大三课程 |

每个课程目录内部遵循松散一致的结构：
- `README.md` — 课程说明（教师、考试建议等）
- `实验/` — 实验任务书、报告、代码
- `课设/` — 课程设计项目
- `考试/` 或 `真题/` 或 `试卷/` — 历年考试真题
- `作业/` — 课后作业及解答
- `PPT/` 或 `上课PPT与教材/` — 课堂幻灯片
- `复习资料/` — 复习笔记总结
- `Zips/` — 大文件的压缩归档

## 网站构建

### 技术栈
- **MkDocs** + **mkdocs-material** 主题构建 GitHub Pages 站点
- 自动部署 CI: `.github/workflows/gh-pages.yml`

### 关键脚本
- `gen-md.py` — 遍历课程目录，为每门课生成 `docs/<课程名>.md`，包含文件列表（链接到 GitHub Raw 内容）
- `mkdocs.yml` — 站点配置（Material 主题、亮暗色模式切换）

### 本地预览
```bash
pip install mkdocs mkdocs-material
python gen-md.py
cp README.md ./docs/index.md
cp -r assets ./docs/assets
mkdocs serve
```

### CI/CD
每次推送至 `main` 分支自动触发 GitHub Actions：
1. 安装 Python + MkDocs
2. 运行 `gen-md.py` 生成文档页面
3. 执行 `mkdocs gh-deploy --force` 发布到 `gh-pages` 分支

## 如何添加新资料

### 目录命名规范

```
【年级】课程名/
├── PPT/             # 课堂幻灯片（可标注新旧版本）
├── 实验/            # 实验任务书、实验代码、实验报告（按实验编号整理）
├── 课设/            # 课程设计项目（含代码、报告）
├── 考试/ 或 真题/   # 历年考试真题（含答案更佳）
├── 作业/            # 课后作业及解答
├── 复习资料/        # 复习笔记、总结
├── 教材and解析/     # 教材 PDF、课后习题答案解析
├── Zips/            # 大文件压缩包
└── README.md        # 课程说明（教师、考试形式、备考建议）
```

### 描述内容的行文规范

在 `README.md` 的资料目录表格中更新课程内容时，遵循以下原则：

1. **具体化** — 不用笼统的"PPT，实验，真题"，而是写明具体情况：
   - PPT 有几套？新旧？多少份？如 `PPT（新旧两套 21 份）`
   - 实验有多少个？用什么语言？如 `实验 8 次（Verilog/SystemVerilog 40+ 文件）`
   - 真题覆盖什么年份范围？如 `真题（合肥+宣城两校区 2012-2024 共 40+ 份）`
   - 有什么特色资源？如 `吴恩达课程，30+ 本电子书`

2. **突出亮点** — 某课程特别丰富的资源要体现出来（如"猴博士课堂笔记"、"LaTeX 源码复习笔记"、"AsmTools 工具包"）

3. **标明教材版本** — 如有教材，尽量写明书名和版本，如 `邱关源电路第 5 版`、`Java 2 实用教程第 5 版`

4. **注明编程语言/工具** — 实验和课设代码注明语言：`C++`、`Java`、`Python/Jupyter`、`Verilog`、`MIPS 汇编`

5. **描述结构化的课程** — 如一门课分多个阶段（如电路分析+模电两部分），在描述中用 `+` 分隔

### 更新资料目录后的后续操作

1. 如果新增了课程目录，需要重新运行 `gen-md.py` 生成对应的文档页面
2. 推送至 `main` 后，CI 会自动重新部署网站
3. 如果要添加大四 `【大四】*` 课程，前缀格式与现有保持一致
